# Agent Instructions

## Semantic Versioning (semver)

This project uses [Semantic Versioning](https://semver.org/). Version numbers follow the format `MAJOR.MINOR.PATCH`.

### When to bump versions

- **PATCH** (e.g. 1.0.6 → 1.0.7): Bug fixes, minor tweaks, dependency updates, cosmetic changes.
- **MINOR** (e.g. 1.0.7 → 1.1.0): New features, new app wrappers, non-breaking enhancements.
- **MAJOR** (e.g. 1.1.0 → 2.0.0): Breaking changes to CLI flags, config format, or install paths.

### Where versions live

Each app has its own version in its `package.json`:
- `apps/outlook/package.json` → `"version": "x.y.z"`
- `apps/teams/package.json` → `"version": "x.y.z"`

The root `package.json` version is for the monorepo itself and does not affect built packages.

### Rules

1. **Every change that will be released MUST include a version bump** in the affected app's `package.json`. Without a bump, APT users will not receive the update.
2. Bump only the apps that were actually changed. If only Outlook changed, only bump `apps/outlook/package.json`.
3. If a change affects shared code in `src/`, bump **both** app versions.
4. Keep the `snapcraft.yaml.template` version in sync when doing major releases.
5. Never skip versions. Always increment from the current value.

### Example

If fixing a bug in the Outlook preload script:
- Open `apps/outlook/package.json`
- Change `"version": "1.0.6"` to `"version": "1.0.7"`
- Commit alongside the fix
